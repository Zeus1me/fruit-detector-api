How It Works
Images are passed through a ResNet-50 Feature Pyramid Network backbone. A Region Proposal Network generates candidate boxes, which are refined via ROI pooling and classified into one of 11 fruit classes.
📊
Training Details
Trained on 3,845 images with 22,757 bounding box annotations from the Kaggle Fruit Object Detection dataset. Achieved 93% avg recall and 84% avg precision after 10 epochs.
💡
What You Get
Per-detection: class label, confidence score, and bounding box coordinates. Plus randomized nutrition facts and interesting trivia for every detected fruit.

05 · ABOUT

About FruitNet

Faster R-CNN microservice for fruit detection — built and trained by Joseph Zeus

Joseph Zeus
Engine trained & created by Joseph Zeus · Northeastern University EAI6010 Applications of AI
Module 5 · 2024
🏗
Architecture
Faster R-CNN with ResNet-50 Feature Pyramid Network. Region Proposal Network + ROI pooling for precise bounding box regression across 11 classes.
📊
Training
10 epochs on 3,845 images with 22,757 annotated boxes. 93% average recall and 84% average precision on 640 validation images.
🚀
Deployment
FastAPI microservice deployed on Hugging Face Spaces. REST endpoint accepts image uploads, returns JSON detections with class, confidence, and bounding boxes.
🍎
Classes
Apple, Tangerine, Pear, Watermelon, Korean Melon, Lemon, Grape, Pineapple, Cantaloupe, Dragon Fruit, Durian — 11 classes total.
⚙️
Tech Stack
PyTorch · Torchvision · FastAPI · Python 3.11 · HF Spaces. Developed in PyCharm on Apple M4. CPU training only — MPS incompatible with Faster R-CNN.
📁
Dataset
Kaggle Fruit Object Detection dataset. YOLO annotations converted to absolute coordinates. Train/Val split with 640 held-out validation images.
